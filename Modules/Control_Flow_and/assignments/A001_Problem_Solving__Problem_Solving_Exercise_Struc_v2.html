Background and Scenario

You are a junior software engineer at a structural consulting firm. Your team has deployed strain-gauge sensors at five nodes along a bridge beam. Each sensor records a load value (in kilonewtons, kN) at regular intervals during a 24-hour monitoring period. You have been given the raw readings for one monitoring session and asked to build an automated analysis tool.

The engineering classification rules your firm uses are as follows:

Classification Load Range (kN)
SafeLess than 150 kN
Caution150 kN to less than 250 kN
Critical250 kN or greater
InvalidNegative values (sensor error)

Your program must process the data, apply these rules, and produce a structured summary.

Step 2 — Validate and Filter Readings

Suggested time: 20–30 minutes

📄 Text Cell — Write this in your notebook before your Step 2 code cell

Approach

I iterate over sensor_readings with a for loop and unpack each tuple into three named variables: sensor_id, node, and load. Using tuple unpacking keeps the code readable and avoids magic index numbers like reading[2].

For each reading I check whether load < 0. If so, the reading is invalid (sensor error). I print a descriptive warning message and then use continue to skip the rest of the loop body for that iteration — this is the correct tool here because I want to move to the next record without executing any further processing on the bad value. If the reading is valid I append the whole tuple to valid_readings.

At the end of the loop I report how many readings were valid and how many were invalid, which lets me sanity-check the results against the dataset I inspected in Step 1 (I can see that sensors 4 and 8 have negative values, so I expect 2 invalid and 13 valid).

💻 Code Cell — Step 2
# Step 2: Validate and filter sensor readings. # Invalid readings are those with a negative load value (sensor error). # 'continue' skips further processing for each invalid record. valid_readings = [] invalid_count = 0 for sensor_id, node, load in sensor_readings: # unpack each tuple if load < 0: # check for invalid reading print(f"WARNING: Sensor {sensor_id} ({node}) recorded an invalid " f"reading of {load} kN. Skipping.") invalid_count += 1 continue # skip to the next iteration valid_readings.append((sensor_id, node, load)) # store valid reading # Report totals print(f"\nValid readings: {len(valid_readings)}") print(f"Invalid readings: {invalid_count}")

Step 3 — Classify Each Valid Reading

Suggested time: 30–40 minutes

📄 Text Cell — Write this in your notebook before your Step 3 code cell

Approach and Data Structure Choices

I iterate over valid_readings using enumerate() so that I have access to a positional index if needed for debugging, while the sensor ID from the tuple serves as the dictionary key.

I apply the classification thresholds with an if / elif / else chain. The order matters: I check for < 150 first (Safe), then < 250 (Caution — which implicitly means ≥ 150 because the first branch did not match), and the final else catches everything ≥ 250 (Critical). This ordering avoids overlapping conditions and is both correct and efficient.

Why a nested dictionary for classification_report? Keying by sensor ID gives O(1) lookup for any specific sensor. The inner dictionary makes each record self-describing — the keys "node", "load_kN", and "status" are explicit labels, which is much clearer than positional indexing into a tuple.

I also maintain three separate flat lists — safe_loads, caution_loads, critical_loads — containing only the numeric load values. These will make the statistical calculations in Step 4 straightforward using min(), max(), and sum().

💻 Code Cell — Step 3
# Step 3: Classify each valid reading and store results. # classification_report: nested dict keyed by sensor_id. # safe_loads / caution_loads / critical_loads: flat lists of numeric values. classification_report = {} safe_loads = [] caution_loads = [] critical_loads = [] for index, (sensor_id, node, load) in enumerate(valid_readings): # Apply classification thresholds in ascending order if load < 150: status = "Safe" safe_loads.append(load) elif load < 250: status = "Caution" caution_loads.append(load) else: # load >= 250 status = "Critical" critical_loads.append(load) # Store the full classified record in the report dictionary classification_report[sensor_id] = { "node": node, "load_kN": load, "status": status, } # Print classification report — one entry per line for readability print("=== Classification Report ===") for sid, record in classification_report.items(): print(f" Sensor {sid:>2} | {record['node']} | " f"{record['load_kN']:>6.1f} kN | {record['status']}")

Step 4 — Generate a Summary Report

Suggested time: 30–45 minutes

📄 Text Cell — Write this in your notebook before your Step 4 code cell

Approach

This step has three distinct sub-tasks, each covered in its own code cell below.

4a — Category Counts and Statistics

For each category list I check whether the list is empty before calling min(), max(), or computing an average. Calling these functions on an empty list raises a ValueError, so the guard is necessary for robustness — even if the current dataset always produces at least one reading per category, a production tool must not crash on edge cases.

4b — Critical Node Identification

I use a set (critical_nodes) to collect node labels whose status is "Critical". A set is the right choice here because the same node (e.g., Node-A) can appear multiple times in the dataset across different time intervals, and I want only unique node names. Sets automatically deduplicate, so I never need to check for membership before adding.

4c — Threshold Scan with Early Exit

I use a for loop with break to scan sensor_readings from the beginning and stop at the first reading that meets or exceeds 300 kN. Using break is more efficient than iterating the entire list when I only need the first match. Python's loop else clause runs only if the loop completes without hitting a break, which is the correct way to handle the "no match found" case — cleaner than a separate boolean flag.

💻 Code Cell — Step 4a: Category Statistics
# Step 4a: Compute and print statistics for each classification category. # Guard against empty lists before calling min(), max(), or computing average. def print_category_stats(label, values): print(f"\n {label}:") if not values: # guard: skip stats if list is empty print(" No readings in this category.") else: print(f" Count: {len(values)}") print(f" Min: {min(values):.1f} kN") print(f" Max: {max(values):.1f} kN") print(f" Average: {sum(values)/len(values):.1f} kN") print("=== Category Statistics ===") print_category_stats("Safe", safe_loads) print_category_stats("Caution", caution_loads) print_category_stats("Critical", critical_loads)
💻 Code Cell — Step 4b: Critical Node Identification
# Step 4b: Collect unique node labels that have at least one Critical reading. # A set is used because nodes can appear multiple times; sets deduplicate automatically. critical_nodes = set() for sid, record in classification_report.items(): if record["status"] == "Critical": critical_nodes.add(record["node"]) print("\n=== Critical Nodes ===") if critical_nodes: print(f" Nodes with at least one Critical reading: {critical_nodes}") else: print(" No critical nodes detected.")
💻 Code Cell — Step 4c: Threshold Scan with Early Exit
# Step 4c: Scan sensor_readings for the FIRST reading >= 300 kN, then stop. # 'break' exits the loop early once a match is found. # The loop's 'else' clause runs only if no break occurred (i.e., no match found). THRESHOLD = 300 # kN print("\n=== Threshold Scan (>= 300 kN) ===") for sensor_id, node, load in sensor_readings: if load >= THRESHOLD: print(f" ALERT: Sensor {sensor_id} ({node}) recorded {load} kN " f"— meets or exceeds {THRESHOLD} kN threshold. Stopping scan.") break else: # Only executes if the loop finished without a 'break' print(f" No readings exceeded {THRESHOLD} kN.")
