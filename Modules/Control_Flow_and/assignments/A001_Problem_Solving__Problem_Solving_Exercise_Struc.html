Purpose

Why This Assignment Matters

Structural engineers routinely analyze large sets of load measurements collected from sensors embedded in bridges, buildings, and other infrastructure. These datasets must be processed automatically — validated for errors, classified by severity, and summarized — because doing so manually at scale is impractical and error-prone.

This assignment simulates exactly that workflow. You will write a Python program that processes a list of simulated load readings from a structural monitoring system, applies engineering classification rules, and produces a summary report. The skills you practice here — conditional logic, iteration, and data structure selection — are foundational to computational engineering, data science, and scientific programming in Python.

This exercise directly supports the module's learning goals by requiring you to integrate control flow constructs and data structures in a realistic, domain-relevant context.

Background and Scenario

You are a junior software engineer at a structural consulting firm. Your team has deployed strain-gauge sensors at five nodes along a bridge beam. Each sensor records a load value (in kilonewtons, kN) at regular intervals during a 24-hour monitoring period. You have been given the raw readings for one monitoring session and asked to build an automated analysis tool.

The engineering classification rules your firm uses are as follows:

| Classification | Load Range (kN) |

|---|---|

| Safe | Less than 150 kN |

| Caution | 150 kN to less than 250 kN |

| Critical | 250 kN or greater |

| Invalid | Negative values (sensor error) |

Your program must process the data, apply these rules, and produce a structured summary.

Task

Overview

You will write a Python script (or Jupyter Notebook) that completes the following five steps. Each step is described in detail below. Read all steps before writing any code.

> Definitions for this assignment:

> - Process means to read each value, apply logic to it, and store or display a result.

> - Filter means to deliberately skip or exclude values that do not meet a criterion (e.g., invalid readings).

> - Classify means to assign each valid reading to exactly one named category based on its numeric value.

> - Summarize means to compute and display aggregate statistics across all readings.

Step 1 — Set Up Your Dataset (Suggested time: 15–20 minutes)

Create the following dataset directly in your Python script as a list of tuples. Each tuple represents one sensor reading in the format (sensor_id, node_label, load_kN).

`

(1, "Node-A", 102.3)

(2, "Node-B", 178.5)

(3, "Node-C", 263.0)

(4, "Node-D", -15.2)

(5, "Node-E", 149.9)

(6, "Node-A", 251.7)

(7, "Node-B", 88.4)

(8, "Node-C", -3.0)

(9, "Node-D", 199.6)

(10, "Node-E", 305.1)

(11, "Node-A", 145.0)

(12, "Node-B", 250.0)

(13, "Node-C", 73.2)

(14, "Node-D", 310.8)

(15, "Node-E", 160.4)

`

Store this as a list of tuples named sensor_readings.

Why tuples? Each individual reading is a fixed record — the sensor ID, node label, and load value form a logically grouped, immutable unit. You should be prepared to briefly explain this choice in your reflection (Step 5).

Step 2 — Validate and Filter Readings (Suggested time: 20–30 minutes)

Using a for loop, iterate over sensor_readings. For each reading:

1. Unpack the tuple into three separate variables (sensor ID, node label, load value).

2. Use a conditional statement to check whether the load value is negative (invalid).

3. If the reading is invalid:

- Print a warning message identifying the sensor ID and node label. Example: WARNING: Sensor 4 (Node-D) recorded an invalid reading of -15.2 kN. Skipping.

- Use continue to skip all further processing for that iteration.

4. If the reading is valid, append it to a new list called valid_readings.

At the end of this step, print the total number of valid readings and the total number of invalid readings detected.

Step 3 — Classify Each Valid Reading (Suggested time: 30–40 minutes)

Using a for loop (with enumerate() to track position), iterate over valid_readings. For each valid reading:

1. Apply the classification rules from the [Background and Scenario](#background-and-scenario) table using if, elif, and else statements.

2. Store the classification result in a dictionary named classification_report. Use the sensor ID as the key and a dictionary as the value. The inner dictionary should contain:

- "node" — the node label string

- "load_kN" — the numeric load value

- "status" — the classification string ("Safe", "Caution", or "Critical")

Example structure:

`python

classification_report = {

1: {"node": "Node-A", "load_kN": 102.3, "status": "Safe"},

...

}

`

3. While iterating, also maintain three separate lists: safe_loads, caution_loads, and critical_loads. Append the load value (not the full tuple) to the appropriate list for each reading.

At the end of this step, print the contents of classification_report in a readable format (one entry per line).

Step 4 — Generate a Summary Report (Suggested time: 30–45 minutes)

After processing all readings, produce a formatted summary report printed to the console. Your report must include all of the following:

#### 4a — Category Counts and Statistics

For each of the three categories (safe_loads, caution_loads, critical_loads):

  • The number of readings in that category
  • The minimum load value in that category (use min())
  • The maximum load value in that category (use max())
  • The average load value in that category (use sum() and len())

Handle the case where a category list is empty using a conditional check before calling min(), max(), or calculating the average. Print "No readings in this category" if the list is empty.

#### 4b — Critical Node Identification

Create a set named critical_nodes. Iterate over classification_report using .items() and add the node label to critical_nodes for every entry whose status is "Critical".

Print the set of critical nodes. Sets are appropriate here because the same node may appear multiple times in the data, and you want only unique node labels.

#### 4c — Threshold Scan with Early Exit

Write a while loop (or a for loop with break) that scans through the original sensor_readings list from the beginning and stops (using break) at the first reading whose load value meets or exceeds 300 kN. Print a message identifying that sensor and its load value. If no such reading exists, use the loop's else clause to print "No readings exceeded 300 kN.".

Step 5 — Written Reflection (Suggested time: 20–30 minutes)

After completing your code, write a short reflection (approximately 200–350 words) addressing all of the following prompts. Your reflection may be included as a comment block at the bottom of your script, as a separate text or Markdown file, or as a cell in your Jupyter Notebook.

1. Process: Briefly describe the steps you took to write and test your program. What did you do first? How did you verify that your output was correct?

2. Data Structure Choices: Why did you use tuples to store raw readings, a dictionary to store the classification report, and a set to store critical nodes? Could you have used different structures? What would have been the trade-offs?

3. Challenges: Describe at least one specific error or unexpected behavior you encountered while writing the code. How did you diagnose and fix it?

4. Learning: What is one concept from this module that became clearer to you through this exercise?
