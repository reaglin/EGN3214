Understanding File Types in Windows

01What a file type actually is

Every file on a computer is just a long sequence of bytes. On its own, that sequence means nothing — it could be a photo, a spreadsheet, or a program. A file type is the agreement about how those bytes should be interpreted. Windows uses that agreement to decide which icon to show, which program to open the file with, and how to treat the contents.

There are really two layers to a file type. The format is the internal structure of the bytes (for example, the rules that make a JPEG a JPEG). The extension is the short label at the end of the filename that advertises which format Windows should expect. Most of the time these two agree — but as you'll see, they don't have to, and that's where things get interesting.

02How extensions identify files

In Windows, the part of a filename after the last dot is the extension. In budget.xlsx, the name is budget and the extension is .xlsx. Windows keeps a master list called the file association table that maps each extension to a default program.

When you double-click report.docx, Windows doesn't inspect the contents. It reads the extension, looks up .docx in the association table, sees that it points to Word, and launches Word with that file. The extension is essentially a routing label.

Key idea

The extension tells Windows what a file claims to be. The actual bytes inside determine what the file really is. A well-behaved file has matching extension and contents. Renaming the extension changes only the label — never the bytes.

03Showing file extensions in File Explorer

By default, Windows hides known extensions, so data.csv may appear simply as data. Before you can reliably change or verify a file type, turn extensions back on. This is the single most useful setting in this module.

  1. Open File Explorer (the folder icon on the taskbar, or press Win + E).
  2. In Windows 11, click the View menu at the top → Show → tick File name extensions.
  3. In Windows 10, click the View tab on the ribbon and tick the File name extensions checkbox.
  4. Extensions now appear on every file. You should see data.csv, not just data.
Why this matters

If extensions are hidden, a file saved by Notepad as data.csv may secretly be data.csv.txt — and you'd never see the extra .txt. Turning extensions on prevents this very common mistake.

04Changing a file type in File Explorer

With extensions visible, changing a file type is just renaming the part after the dot. This is how you would, for instance, turn a plain text file into a CSV.

  1. Select the file once (single click) so it's highlighted but not opened.
  2. Press F2, or right-click and choose Rename.
  3. Edit the extension after the dot — e.g. change data.txt to data.csv. Leave the name before the dot alone unless you also want to rename it.
  4. Press Enter. Windows warns: “If you change a file name extension, the file might become unusable.” Click Yes only if you're sure the contents truly match the new type.
Important

Renaming only works as a true conversion when the file's internal bytes already match the target format. Plain text genuinely is the right format for both .txt and .csv, so renaming between them is safe. Renaming a .png to .jpg is not safe — see the next section.

05Side effects of changing a file type

Because renaming changes only the label and never the bytes, mismatching the two has consequences. Here is what can go wrong.

The file may become unreadable

If you rename photo.png to photo.jpg, the bytes are still in PNG format. An image viewer that trusts the .jpg label tries to decode JPEG data, fails, and either shows a corrupt image or an error. The file isn't damaged — the label is simply lying about it.

It opens in the wrong program

Changing the extension reroutes the file to a different default program. Rename a spreadsheet's .xlsx to .txt and double-clicking now launches Notepad, which displays the raw, unreadable internal structure of the spreadsheet instead of a tidy grid.

Real conversion needs real software

To genuinely change a format — PNG to JPG, DOCX to PDF, XLSX to CSV — you must open the file in a program and use Save As / Export. That re-encodes the bytes into the new format. Renaming never does this.

The safe exception

Plain-text formats are interchangeable by renaming because they share identical byte structure. .txt, .csv, .log, .md, and many code files are all just text. Renaming among these is harmless — which is exactly why the Notepad-to-CSV trick below works.

07Assignment: Create & save a CSV with Notepad

A CSV is simply plain text where each line is a row and commas separate the columns. Because it's plain text, you can build one by hand in Notepad — no Excel required. Follow these steps exactly.

  1. Open Notepad. Click Start, type Notepad, press Enter.
  2. Type your data with commas between values and one record per line. For example: 
    name,age,city
Ada,36,London
Grace,41,New York
Linus,28,Helsinki
  3. Open Save As. Go to File → Save As (or press Ctrl + Shift + S).
  4. Set "Save as type" to All Files (*.*). This stops Notepad from forcing a .txt ending onto your filename.
  5. Name it with the .csv extension — type students.csv exactly, including the extension, in the File name box.
  6. Set Encoding to UTF-8 in the dropdown at the bottom of the dialog, then click Save.
  7. Verify. In File Explorer (with extensions shown), confirm the file reads students.csv and not students.csv.txt.
The #1 mistake

If you leave "Save as type" on Text Documents (*.txt), Notepad appends .txt and you get students.csv.txt. With extensions hidden you won't notice. Always choose All Files first — this is why Section 03 had you turn extensions on.

Rules for valid CSV

Use a plain comma (not a semicolon) as the separator. Don't add spaces after commas unless they're part of the value. Every row should have the same number of columns. If a value itself contains a comma, wrap that value in double quotes, e.g. "Smith, John".

08Alternative: Host the CSV on Google Drive for Colab

If you'd rather not work locally in Windows, you can keep the same CSV in Google Drive and open it directly from Google Colab. Drive can host any file type — CSVs, images, notebooks, datasets — and Colab can read them once Drive is connected.

Step 1 — Get the CSV into Drive

You can create the file the same way as above and then upload it, or build it in the browser:

  • Upload an existing file: open drive.google.com, click New → File upload, and select your students.csv.
  • Or make one in Sheets: create a Google Sheet, enter your data, then File → Download → Comma-separated values (.csv). (Note that the Sheet itself is not a CSV — you must export it.)

Step 2 — Mount Drive in Colab

In a Colab notebook, run this cell. It will prompt you to authorize access to your Drive.

# Connect your Google Drive to this Colab session
from google.colab import drive
drive.mount('/content/drive')

Step 3 — Read the CSV

Once mounted, your Drive lives under /content/drive/MyDrive/. Point pandas at the file using its path:

# Adjust the path to wherever you saved the file in Drive
import pandas as pd
df = pd.read_csv('/content/drive/MyDrive/students.csv')
df.head()
Finding the path

If the file sits in a folder, include it: /content/drive/MyDrive/Module4/students.csv. In Colab's left sidebar, open the Files panel, browse into drive/MyDrive, right-click the file, and choose Copy path to get it exactly right.

Either path is fine

The Notepad method and the Drive method produce the same kind of file — a plain-text CSV. Use Notepad if you're working on a Windows machine and want to understand file types hands-on; use Drive if you'd prefer to stay in the browser and load straight into Colab.
