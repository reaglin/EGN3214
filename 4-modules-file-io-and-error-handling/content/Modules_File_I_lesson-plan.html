Modules_File_I

Learning Objectives

  • Explain the purpose of file I/O and describe the standard file modes (r, w, a, rb, wb, etc.) used when opening files in Python
  • Demonstrate how to open and close files safely using the with statement as a context manager
  • Read data from text files using methods such as read(), readline(), and readlines()
  • Write and append data to text files using write() and writelines()
  • Construct and manipulate file system paths programmatically using the pathlib module (Path objects, path concatenation, existence checks, and directory listing)
  • Apply best practices for file handling including exception awareness and cross-platform path compatibility

Prerequisites

  • Completion of prior EGN3214 modules covering Python fundamentals (variables, data types, operators)
  • Working knowledge of Python control flow: if/elif/else, for and while loops
  • Familiarity with Python functions: defining, calling, and passing arguments
  • Understanding of Python lists, strings, and basic string methods
  • Basic experience using an IDE or text editor (VS Code, PyCharm, or Jupyter Notebook) and running Python scripts from the command line or terminal

Instructional Activities

  • Activity 1 — Hook / Real-World Motivation (10 min): Instructor presents brief examples of real engineering applications that rely on file I/O (reading sensor data logs, writing simulation output, parsing CSV configuration files). Class discussion: "Where have you encountered data stored in files in your own projects?"
  • Activity 2 — Direct Instruction: Opening Files and File Modes (15 min): Instructor explains the built-in open() function, introduces all standard file modes (r, w, a, x, b, t, +), and discusses what happens when a file does not exist. Live demo: open a file in each common mode and observe behavior.
  • Activity 3 — Direct Instruction: The with Statement Context Manager (15 min): Instructor explains resource management, the concept of a context manager, and how the with statement guarantees file closure even when exceptions occur. Compare with/without the with statement side-by-side. Highlight __enter__ and __exit__ at a conceptual level.
  • Activity 4 — Guided Practice: Reading Text Files (20 min): Instructor live-codes reading a provided sample engineering data file using read(), readline(), and readlines(). Students follow along in their own environment. Discuss trade-offs of each method (memory usage, iteration patterns). Brief pair-and-share: students modify the code to count lines and words.
  • Activity 5 — Guided Practice: Writing Text Files (20 min): Instructor demonstrates writing new content to a file with mode 'w' and appending with mode 'a'. Cover write() vs writelines(). Students complete a short exercise: generate a simple formatted report (student name, ID, computed value) and save it to a .txt file.
  • Activity 6 — Direct Instruction + Demo: Working with pathlib (20 min): Introduce the pathlib.Path class; demonstrate creating Path objects, joining paths with /, checking existence (.exists(), .is_file(), .is_dir()), listing directory contents (.iterdir()), and getting file metadata (.stem, .suffix, .name, .parent). Emphasize cross-platform advantages over os.path string manipulation.
  • Activity 7 — Independent Lab Exercise (25 min): Students complete a structured lab: (1) Use pathlib to locate all .txt files in a given directory. (2) Read each file, extract numeric values on each line, and compute the average. (3) Write a summary report file containing the filename and computed average for each input file. Instructor circulates and provides targeted feedback.
  • Activity 8 — Debrief and Q&A (10 min): Instructor reviews common errors observed during the lab (FileNotFoundError, incorrect modes, path separators). Reinforce best practices. Open Q&A to address remaining conceptual questions.

Assessment / Check for Understanding

  • Entrance Ticket (pre-activity, ungraded): A 2-question poll on the LMS asking students to identify what a file mode does and where they have seen file I/O used — used to gauge prior exposure and adjust pacing
  • Think-Pair-Share (during Activity 4): Students predict the output of a code snippet that reads a file with readline() in a loop; pairs compare answers before class discussion
  • Exit Ticket (end of class, ungraded): Three short written questions — (1) What is the primary advantage of using the with statement over manually calling file.close()? (2) Write the one-line Python statement to open data.txt for appending. (3) Write a pathlib expression to check whether a file named results.txt exists in the current directory
  • Lab Report (graded, individual submission via LMS): Students submit their completed lab script from Activity 7 along with a generated summary report file. Rubric evaluates correctness of pathlib usage (25%), accurate file reading and data extraction (25%), correct writing of the summary report (25%), and code readability/comments (25%)
  • Module Quiz (graded, auto-graded on LMS — 10 multiple-choice and 2 short-answer questions): Covers all five topic areas — file modes, with statement behavior, reading methods, writing methods, and pathlib operations. Short-answer items ask students to write small code snippets; administered within one week following the lesson
