Module Topics & Outcomes

Module 1: 1 - Python Environment Setup and Language Fundamentals

Topics

Python Programming Environments

Covers the tools and interfaces used to write and run Python code, including IDEs and Jupyter notebooks. Learners will understand how to choose an appropriate environment for different tasks.
  • What Is a Python Environment? — A Python programming environment is the combination of tools, interpreters, and interfaces that allow a developer to write, run, and debug Python code.
  • Integrated Development Environments (IDEs) — An IDE is a software application that provides comprehensive tools for writing and debugging code, typically including a code editor, debugger, and project manager in one interface.
  • Jupyter Notebooks — Jupyter Notebook is an open-source, browser-based environment that allows you to write and execute Python code in discrete, runnable cells alongside narrative text and visualizations.
  • The Python Interpreter and REPL — The Python interpreter is the program that reads and executes Python code. When launched interactively, it provides a Read-Eval-Print Loop (REPL) where each line of code is executed immediately.

Package Management in Python

Explores how Python packages extend the language's capabilities and how to install, manage, and organize them using tools like pip and virtual environments.
  • What Are Python Packages? — A Python package is a collection of modules that provide pre-written code for specific tasks, allowing developers to reuse functionality without writing it from scratch.
  • Using pip to Install Packages — pip is the default package manager for Python and allows users to install, upgrade, and remove packages from the command line.
  • Virtual Environments — A virtual environment is an isolated Python environment that allows different projects to have their own dependencies without conflicting with one another.

Interactive vs. Script Execution

Distinguishes between running Python code interactively in a REPL or notebook and executing standalone Python script files. Learners will understand when to use each approach.
  • Interactive Execution — Interactive execution allows a programmer to type Python statements one at a time and see results immediately, making it ideal for learning, testing, and data exploration.
  • Script Execution — A Python script is a plain-text file with a .py extension that contains a sequence of Python statements executed from top to bottom when the file is run.
  • Choosing the Right Execution Mode — Selecting between interactive and script execution depends on the goal: interactive mode suits exploration and learning, while script mode suits repeatable, structured programs.

Variables and Assignment

Introduces how Python uses variables to store and reference data, covering naming conventions, assignment syntax, and how Python manages variable references in memory.
  • What Is a Variable? — A variable is a named reference that points to a value stored in memory, allowing programmers to store, retrieve, and manipulate data throughout a program.
  • Naming Rules and Conventions — Python enforces specific rules for variable names and follows community conventions documented in PEP 8 to promote readable, consistent code.
  • Multiple Assignment and Unpacking — Python supports assigning multiple variables at once, either to the same value or to individual values from a sequence, using concise syntax.

Core Data Types

Covers Python's fundamental built-in data types including integers, floats, strings, and booleans, explaining their characteristics and appropriate use cases.
  • Integers and Floating-Point Numbers — Python provides two primary numeric types: integers (int) for whole numbers and floating-point numbers (float) for decimals and numbers requiring fractional precision.
  • Strings — A string (str) is an ordered, immutable sequence of characters used to represent text in Python, enclosed in single or double quotation marks.
  • Booleans — The boolean (bool) type represents one of two truth values — True or False — and is fundamental to controlling program flow through conditionals and loops.
  • The None Type — None is a special singleton value in Python that represents the absence of a value or a null result, commonly used as a default or placeholder.
  • Checking Data Types with type() — The built-in type() function returns the type of any Python object, allowing programmers to inspect and verify the data type of a variable at runtime.

Operators in Python

Introduces the main categories of Python operators — arithmetic, comparison, logical, and assignment — explaining their syntax, behavior, and precedence rules.
  • Arithmetic Operators — Arithmetic operators perform mathematical calculations on numeric values, including addition, subtraction, multiplication, division, and more specialized operations.
  • Comparison Operators — Comparison operators evaluate the relationship between two values and return a boolean result, forming the basis of conditional logic in Python programs.
  • Logical Operators — Logical operators — and, or, and not — combine or invert boolean expressions and are central to building complex conditional statements.
  • Assignment Operators — Assignment operators store a value in a variable, with augmented assignment operators providing a shorthand for updating a variable's value in place.
  • Operator Precedence — Python evaluates expressions according to a defined order of precedence, determining which operations are performed first when multiple operators appear in a single expression.

Type Conversion

Explains how Python handles conversion between data types, distinguishing between implicit coercion performed automatically by Python and explicit conversion performed by the programmer.
  • Implicit Type Conversion — Implicit type conversion, or coercion, occurs automatically when Python converts one data type to another to complete an operation without requiring any action from the programmer.
  • Explicit Type Conversion with Built-in Functions — Explicit type conversion, also called casting, uses built-in functions such as int(), float(), str(), and bool() to deliberately convert a value from one type to another.
  • Common Conversion Errors and How to Avoid Them — Incorrect type conversion attempts raise exceptions in Python, so understanding which conversions are valid and how to handle failures is an essential programming skill.

Module 2: 2 - Control Flow and Python Data Structures

Topics

Conditional Statements

Conditional statements allow a program to make decisions by executing different code blocks based on whether a condition evaluates to true or false. This topic covers if, elif, and else constructs in Python.
  • The if Statement — The if statement is the most fundamental conditional construct in Python, executing a block of code only when a specified condition is True.
  • The elif Clause — The elif (else-if) clause allows a program to check multiple conditions in sequence after an initial if condition fails.
  • The else Clause — The else clause defines a default block of code that executes when none of the preceding if or elif conditions are True.
  • Comparison and Logical Operators — Conditions in Python are built using comparison operators (==, !=, <, >, <=, >=) and logical operators (and, or, not) to form complex Boolean expressions.
  • Nested Conditionals — Conditional statements can be placed inside other conditional blocks to test additional conditions when an outer condition is True.

for Loops and the range() Function

The for loop iterates over a sequence of items, making it ideal for processing collections of data. The range() function generates numeric sequences commonly used to control loop iterations.
  • The for Loop — A for loop in Python iterates over each item in a sequence — such as a list, string, or range — executing the loop body once per item.
  • The range() Function — The range() function generates an immutable sequence of integers and is commonly used with for loops to repeat an action a specific number of times.
  • Iterating with an Index Using enumerate() — The enumerate() function allows a for loop to access both the index and the value of each item in an iterable simultaneously.
  • Looping Over Numeric Ranges in Engineering — Engineering applications frequently require iterating over a defined numeric range, such as time steps in a simulation or indices in a data array.

while Loops

A while loop repeats a block of code for as long as a specified condition remains True, making it suitable for situations where the number of iterations is not known in advance.
  • The while Loop Structure — A while loop continuously executes its body until its controlling condition evaluates to False, checking the condition before each iteration.
  • Infinite Loops and Loop Guards — An infinite loop occurs when a while loop's condition never becomes False, causing the program to run indefinitely without halting.
  • Comparing for and while Loops — Choosing between a for loop and a while loop depends on whether the number of iterations is known in advance or determined dynamically during execution.

Loop Control: break and continue

The break and continue statements provide fine-grained control over loop execution, allowing early termination or skipping specific iterations without exiting the loop entirely.
  • The break Statement — The break statement immediately terminates the innermost loop it appears in, transferring control to the first statement after the loop.
  • The continue Statement — The continue statement skips the remainder of the current loop iteration and jumps directly to the next iteration's condition check.
  • The else Clause on Loops — Python allows an optional else clause on both for and while loops, which executes only if the loop completed normally without encountering a break statement.
  • Practical Use Cases for break and continue — In engineering and data processing, break and continue are frequently used to handle exceptions in data, implement early-exit optimizations, and process conditional sequences.

Lists

Lists are ordered, mutable sequences that can hold items of any type, making them one of the most versatile and widely used data structures in Python.
  • Creating and Accessing Lists — A list is created using square brackets and can store any mix of data types. Individual elements are accessed using zero-based integer indices.
  • Modifying Lists — Because lists are mutable, their elements can be changed, added, or removed after the list is created using built-in methods and direct assignment.
  • List Slicing — Slicing extracts a sub-list from a list using the syntax list[start:stop:step], returning a new list containing the selected elements.
  • List Comprehensions — A list comprehension provides a concise, readable way to create a new list by applying an expression to each item in an iterable, optionally filtered by a condition.
  • Common List Operations and Methods — Python provides a rich set of built-in functions and list methods for sorting, searching, counting, and transforming list data.

Tuples

Tuples are ordered, immutable sequences that are ideal for storing fixed collections of data that should not be changed after creation.
  • Creating and Accessing Tuples — Tuples are defined using parentheses (or simply commas) and support the same indexing and slicing operations as lists, but their contents cannot be modified.
  • Immutability of Tuples — Once a tuple is created, its elements cannot be added, removed, or changed, making tuples a safer choice when data integrity is important.
  • Tuple Packing and Unpacking — Tuple packing assigns multiple values into a single tuple, while tuple unpacking assigns the elements of a tuple to individual variables in one statement.
  • When to Use Tuples vs. Lists — Tuples are preferred when the data represents a fixed record or should be protected from modification, while lists are better suited for collections that need to grow or change.

Dictionaries

Dictionaries store data as key-value pairs, enabling fast lookups by key rather than by position, and are ideal for representing structured, labeled data.
  • Creating and Accessing Dictionaries — Dictionaries are created using curly braces with colon-separated key-value pairs. Values are retrieved by supplying the corresponding key in square brackets.
  • Modifying Dictionaries — Dictionaries are mutable; entries can be added, updated, or deleted at any time using direct assignment and built-in methods.
  • Iterating Over Dictionaries — Python provides dedicated methods for iterating over a dictionary's keys, values, or key-value pairs using keys(), values(), and items().
  • Engineering Applications of Dictionaries — Dictionaries are well suited for mapping labeled engineering data, such as material properties, configuration parameters, or named measurement channels.

Sets and Choosing the Right Data Structure

Sets store unique, unordered elements and support efficient membership testing and set operations. This topic also provides guidance on selecting the appropriate data structure for engineering problems.
  • Creating and Using Sets — A set is an unordered collection of unique elements defined with curly braces or the set() constructor. It automatically discards duplicate values.
  • Set Operations — Sets support mathematical operations including union, intersection, difference, and symmetric difference, enabling powerful comparisons between groups of data.
  • Membership Testing with Sets — Sets provide O(1) average-time membership testing using the in operator, making them significantly faster than lists for checking whether a value exists in a large collection.
  • Comparing Core Data Structures — Lists, tuples, dictionaries, and sets each have distinct properties — ordered vs. unordered, mutable vs. immutable, indexed vs. keyed — that make them appropriate for different tasks.
  • Selecting the Right Structure for Engineering Problems — Choosing the appropriate data structure improves program correctness, performance, and readability, particularly in data-intensive engineering applications.

Module 3: 3 - Functions, Comprehensions, and Functional Patterns

Topics

Defining Functions

Introduces the syntax and mechanics of creating reusable functions in Python. Covers the def keyword, naming conventions, and the role of the return statement.
  • The def Statement — Functions in Python are defined using the def keyword followed by a name and a parenthesized parameter list. The indented block beneath the signature forms the function body.
  • The return Statement — The return statement sends a value back to the caller and immediately exits the function. A function without an explicit return statement implicitly returns None.
  • Docstrings and Function Documentation — A docstring is a string literal placed as the first statement inside a function body and serves as built-in documentation. It is accessible at runtime via the function's __doc__ attribute or the help() function.
  • Calling a Function — A function is invoked by writing its name followed by parentheses containing any required arguments. The interpreter jumps to the function body, executes it, and resumes at the call site with the returned value.

Parameters and Arguments

Explores the different ways values can be passed into functions, including positional, keyword, and default parameters. Establishes clear terminology between parameters and arguments.
  • Positional Parameters — Positional parameters are matched to arguments in the order they appear in the function signature. The first argument maps to the first parameter, the second to the second, and so on.
  • Keyword Arguments — Keyword arguments are passed by explicitly naming the parameter they correspond to, allowing arguments to be supplied in any order. This improves readability, especially in functions with many parameters.
  • Default Parameter Values — A parameter can be assigned a default value in the function signature, making the argument optional at the call site. If the caller does not supply that argument, the default is used.
  • Parameters vs. Arguments Terminology — Parameters are the variable names listed in the function definition, while arguments are the actual values passed when the function is called. Keeping this distinction clear helps in reading error messages and documentation.

Variable-Length Arguments: *args and **kwargs

Covers the special *args and **kwargs syntax that allows functions to accept an arbitrary number of positional or keyword arguments. Explains how these mechanisms support flexible and reusable API design.
  • Collecting Positional Arguments with *args — Prefixing a parameter name with an asterisk (*) causes Python to collect all extra positional arguments into a tuple assigned to that parameter. By convention the name args is used, though any valid identifier works.
  • Collecting Keyword Arguments with **kwargs — Prefixing a parameter name with two asterisks (**) causes Python to collect all extra keyword arguments into a dictionary. The keys are the argument names and the values are the supplied values.
  • Combining *args and **kwargs — A function can declare both *args and **kwargs together to accept any combination of positional and keyword arguments. This pattern is common in decorators and generic wrapper functions.
  • Unpacking Arguments at the Call Site — The * and ** operators can also be used at the call site to unpack a list or tuple into positional arguments and a dictionary into keyword arguments respectively. This is the inverse of the collection behavior in definitions.

Scope and Variable Visibility

Explains how Python resolves variable names using the LEGB rule and how the global and nonlocal keywords allow functions to modify variables in enclosing scopes.
  • The LEGB Scope Rule — Python resolves names by searching four scopes in order: Local, Enclosing, Global, and Built-in. The first scope in which the name is found wins.
  • Local Variables — Any name assigned inside a function is treated as local to that function by default, even if a global variable with the same name exists. Local variables do not affect the outer scope.
  • The global Keyword — The global keyword declares that a name inside a function refers to a module-level variable rather than creating a new local one. This allows the function to read and modify the global variable.
  • The nonlocal Keyword — The nonlocal keyword is used inside a nested function to refer to a variable in the nearest enclosing function scope that is not global. It enables inner functions to modify variables from their enclosing scope.

Recursion

Introduces recursion as a technique where a function calls itself to solve a problem by reducing it to smaller instances. Covers base cases, recursive cases, and practical considerations like stack depth.
  • What Is Recursion? — Recursion is a programming technique in which a function calls itself as part of its own definition. It is a natural fit for problems that can be broken into smaller versions of the same problem.
  • Base Cases — A base case is a condition under which the recursive function returns a result directly without making another recursive call. Every recursive function must have at least one base case to eventually terminate.
  • Recursive Cases — The recursive case defines how the function calls itself with a modified argument that moves closer to the base case. The result of the recursive call is typically combined with additional computation to produce the answer.
  • Recursion Depth and Limitations — Python enforces a maximum recursion depth, defaultable around 1000 calls, to prevent call stack overflow. Very deep recursion will raise a RecursionError even when the logic is correct.

Comprehensions

Teaches the concise comprehension syntax for building lists, dictionaries, and sets from iterables. Covers optional filtering conditions and nested comprehensions.
  • List Comprehensions — A list comprehension provides a compact syntax for creating a new list by applying an expression to each item in an iterable. It replaces many common for-loop patterns with a single readable line.
  • Dictionary Comprehensions — Dictionary comprehensions build a new dictionary by specifying a key-value expression iterated over a source iterable. They are the dict equivalent of list comprehensions.
  • Set Comprehensions — Set comprehensions use curly braces without a colon to produce a set of unique values derived from an iterable. Duplicate expressions are automatically discarded because sets do not allow repeated elements.
  • Filtering in Comprehensions — All three comprehension types support an optional if clause that acts as a filter, including only items for which the condition evaluates to True. Filtering replaces a nested if statement inside a for loop.
  • Nested Comprehensions — Comprehensions can contain multiple for clauses to iterate over nested iterables, effectively flattening or combining data from multiple sequences. The leftmost for clause is the outermost loop.

Lambda Functions and Functional Patterns

Introduces anonymous lambda functions and the map and filter built-ins as tools for applying functional programming patterns in Python. Explains when to use these tools versus comprehensions or named functions.
  • Lambda Functions — A lambda function is an anonymous function defined with the lambda keyword in a single expression. It is used when a small throwaway function is needed without the overhead of a full def statement.
  • The map() Function — The built-in map() applies a function to every item in an iterable and returns a map object, a lazy iterator, that yields the transformed values. It is the functional equivalent of a transformation comprehension.
  • The filter() Function — The built-in filter() applies a predicate function to each element of an iterable and returns only the elements for which the predicate returns True. It is the functional equivalent of a comprehension with an if clause.
  • Combining map and filter — map() and filter() can be composed by nesting calls, applying filtering first and then transformation, or vice versa. This chains functional operations without creating intermediate lists.
  • Choosing Between Comprehensions and Functional Tools — Python offers both comprehensions and map/filter as ways to transform and filter data. Choosing between them depends on readability, the existence of a named function, and team conventions.

Module 4: 4 - Modules, File I/O, and Error Handling

Topics

Introduction to Python Modules

Explains what modules are, why they are used, and how Python locates and loads them at runtime.
  • What Is a Module? — A module is a single Python file containing definitions, functions, classes, and variables that can be reused in other scripts.
  • Importing Modules with import — The import statement loads a module into the current namespace, making its contents accessible via dot notation.
  • Selective Imports with from…import — The from…import syntax lets you import specific names from a module directly into the current namespace.
  • Module Aliases — The 'as' keyword creates a shorter or more descriptive alias for an imported module or name.
  • The Module Search Path — When Python encounters an import statement it searches a sequence of directories defined in sys.path to locate the module file.

Writing and Using Custom Modules

Covers how to design, write, and import your own Python modules to share reusable engineering functions across projects.
  • Creating a Custom Module File — A custom module is created by writing functions, classes, or constants in a .py file and saving it in a location accessible to your scripts.
  • The __name__ == '__main__' Guard — This guard allows a module file to contain both reusable code and a standalone test or demo block that only executes when the file is run directly.
  • Organizing Functions and Constants — Group related functions and constants logically within a module to make the interface clear to other developers and scripts.
  • Importing Your Custom Module — Once saved in the correct directory, a custom module is imported using the same syntax as any standard library module.
  • Documenting a Custom Module — Good documentation within a module enables other engineers to understand and correctly use its functions without reading the implementation.

Using Standard Library Modules

Introduces key modules included with Python that are immediately useful for engineering scripting tasks without additional installation.
  • The math Module — The math module provides mathematical functions and constants commonly needed in engineering calculations.
  • The os and os.path Modules — The os module provides operating-system-independent functions for interacting with the file system and environment.
  • The sys Module — The sys module exposes interpreter-level variables and functions, including command-line arguments and the module search path.
  • The datetime Module — The datetime module supplies classes for manipulating dates and times, which is useful for timestamping engineering data logs.
  • The random Module — The random module generates pseudo-random numbers and provides tools for random sampling, useful in simulations and testing.

File I/O with the with Statement

Explains how to open, read, and write text files safely using Python's with-statement context manager to guarantee proper resource cleanup.
  • Opening Files and File Modes — Python's built-in open() function returns a file object that provides methods for reading and writing; the mode argument determines how the file is accessed.
  • The with Statement Context Manager — The with statement ensures a file is automatically closed when the indented block exits, even if an exception occurs.
  • Reading Text Files — File objects provide multiple methods for reading text content depending on whether you need the entire file, one line, or all lines as a list.
  • Writing Text Files — Opening a file in write or append mode allows you to output strings to the file using the write or writelines methods.
  • Working with File Paths Using pathlib — The pathlib module provides an object-oriented interface for constructing and manipulating file paths that works consistently across operating systems.

Reading and Writing CSV Files

Covers using Python's built-in csv module to read tabular data from CSV files and write structured results, a common task in engineering data workflows.
  • What Is a CSV File? — A CSV (Comma-Separated Values) file stores tabular data as plain text, with each row on a new line and columns separated by a delimiter, typically a comma.
  • Reading CSV Files with csv.reader — csv.reader wraps a file object and returns an iterator that yields each row as a list of strings.
  • Reading CSV Files with csv.DictReader — csv.DictReader reads each row as an ordered dictionary, using the header row as keys, which makes column access by name more readable.
  • Writing CSV Files with csv.writer — csv.writer provides methods to write rows to a CSV file, handling quoting and delimiter escaping automatically.
  • Writing CSV Files with csv.DictWriter — csv.DictWriter writes dictionaries as CSV rows, with field names specified at construction time.

Working with JSON

Introduces JSON as a human-readable data interchange format and demonstrates how Python's json module serializes and deserializes data for configuration files and APIs.
  • What Is JSON? — JSON (JavaScript Object Notation) is a lightweight, text-based format for representing structured data using key-value pairs, arrays, and nested objects.
  • Parsing JSON with json.loads and json.load — Python's json module converts a JSON string or file into native Python data structures using the loads and load functions.
  • Serializing Data with json.dumps and json.dump — The json module converts Python objects into JSON-formatted strings or writes them to a file using dumps and dump.
  • JSON for Configuration Files — Storing simulation parameters or tool settings in a JSON file allows engineers to modify inputs without editing source code.
  • Limitations of Python's json Module — Not all Python objects can be directly serialized to JSON; certain types require custom handling or conversion before they can be encoded.

Error Handling with try-except

Explains Python's exception handling model and demonstrates how to use try-except blocks to build resilient engineering scripts that recover gracefully from runtime errors.
  • What Is an Exception? — An exception is a runtime error that interrupts normal program flow; if not handled, it terminates the script and displays a traceback.
  • Basic try-except Structure — A try block contains code that might raise an exception; the except block defines how to respond if a specific exception type is raised.
  • The else and finally Clauses — The optional else clause runs only if no exception was raised; the finally clause runs unconditionally and is used for cleanup actions.
  • Accessing Exception Information — Using 'as' in an except clause binds the exception object to a variable, providing access to its message and attributes for logging or debugging.
  • Raising Exceptions — The raise statement lets you deliberately trigger an exception when your code detects an invalid condition, signaling errors clearly to calling code.
  • Practical Error Handling in Engineering Scripts — Combining try-except with file I/O, user input, and numeric operations produces robust scripts that handle real-world data quality issues gracefully.

Module 5: 5 - NumPy for Engineering Numerical Computation

Topics

Numpy Functions

A summary of Numpy functions
  • Mathematical and Arithmetic Functions — NumPy provides a rich set of universal functions (ufuncs) for performing element-wise mathematical operations on arrays. These include basic arithmetic as well as advanced mathematical computations commonly used in engineering analysis.
  • Statistical Functions — NumPy includes a comprehensive suite of statistical functions that allow engineers to analyze datasets, compute descriptive statistics, and understand data distributions. These functions operate directly on arrays and support axis-based computations.
  • Linear Algebra Functions — NumPy's numpy.linalg module provides essential linear algebra operations used extensively in structural analysis, control systems, and signal processing. These functions handle matrix operations that are fundamental to solving engineering problems.
  • Array Manipulation Functions — NumPy offers a wide range of functions for reshaping, joining, splitting, and transforming arrays, enabling engineers to prepare and restructure data for computational workflows. These functions are essential for adapting data to the required format of algorithms and simulations.
  • Sorting and Searching Functions — NumPy provides functions to sort array elements and search for values meeting specific criteria, which are frequently needed in data preprocessing and engineering signal analysis. These functions support multi-dimensional arrays and offer control over sorting algorithms.

Time-Stepping Simulation Fundamentals

Introduces the principles of numerical time-stepping methods used to simulate dynamic engineering systems governed by ordinary differential equations, with emphasis on implementation in Python.
  • Ordinary Differential Equations in Engineering — Many engineering systems are governed by ordinary differential equations (ODEs) that describe how state variables such as position, current, or temperature change over time.

  • Runge-Kutta Methods — Runge-Kutta methods, particularly the classical fourth-order RK4 scheme, provide substantially higher accuracy than Euler's method for the same step size.
  • Implementing a Time-Stepping Loop — A time-stepping simulation requires a structured loop that advances the system state from initial conditions to the final simulation time using the chosen numerical method.

Simulating Mechanical Systems

  • Underdamped, Critically Damped, and Overdamped Responses — The damping ratio determines three qualitatively different responses to initial displacement or applied force that have distinct engineering implications.
